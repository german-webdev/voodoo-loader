# Voodoo Loader: Функциональное описание и план миграции на Tauri v2

Обновлено: 2026-03-28
Статус: draft baseline

## 1. Текущая реализация (as-is)

### 1.1 Основная цель продукта
Voodoo Loader — desktop-приложение для массовых и одиночных загрузок файлов через `aria2c` с приоритетом на:
- стабильную очередь,
- управление скоростью/параллелизмом,
- удобный UX для авторизованных источников,
- portable-дистрибуцию.

### 1.2 Основной экран (по текущему UI)
- Верхнее меню: `File`, `Downloads`, `View`, `Settings`, `Help`.
- Ввод URL и добавление в очередь.
- Блок `Destination`:
  - путь сохранения,
  - выбор папки (`Browse`),
  - optional имя файла для одиночной загрузки.
- Блок `Speed`:
  - пресеты скорости,
  - параметры `aria2` в настройках.
- Блок `Progress`:
  - общий прогресс активной сессии,
  - раскрытие подробностей (`More/Less`).
- Управление сессией:
  - `Start`, `Stop`, `Preview command`, `Clear log`.
- Блок `Download queue`:
  - таблица очереди,
  - чекбоксы выбора,
  - drag-and-drop reorder,
  - статусы/прогресс/скорость/ETA/размер/приоритет.
- Блок `Logs`:
  - диагностические и операционные сообщения.

### 1.3 Поддерживаемые функции
- Очередь загрузок:
  - добавление URL по одному,
  - импорт батч-списка из `.txt`,
  - сортировки и приоритизация,
  - retry/remove по выбранным и по failed/canceled.
- Авторизация:
  - `No auth`,
  - `Token + headers`,
  - `Login/password`.
- Безопасность:
  - маскирование чувствительных данных в preview/logs,
  - хранение секретов только по opt-in (`Remember token/username`).
- Локализация:
  - RU/EN с переключением в приложении.
- Звуковые уведомления:
  - start/success/failure.
- Обновление приложения:
  - проверка GitHub Releases,
  - загрузка и применение обновления,
  - попытка автоперезапуска.
- Сборка/доставка:
  - portable сборки,
  - CI/CD pipeline с QA gate, auto-tag и release artifacts.

## 2. Ключевые проблемы текущей платформы

- UI на Qt/PySide6 ограничивает скорость итераций по современному UX.
- Сложности и инциденты вокруг self-update (особенно Windows runtime hygiene).
- Повышенная стоимость поддержки кроссплатформенной упаковки на текущем стеке.
- Больше ручной/скриптовой инфраструктуры, чем хотелось бы для стабильного desktop lifecycle.

## 3. Цель миграции (to-be)

Сделать кроссплатформенное приложение на `Tauri v2` с современным интерфейсом, сохранив всю текущую бизнес-функциональность.

Целевые платформы:
- Windows x64 (первый приоритет),
- Linux x64,
- macOS (после стабилизации core-функций).

## 4. Целевая архитектура на Tauri v2

### 4.1 Frontend
- `TypeScript + React + Vite` (базовый вариант).
- Дизайн-система (токены, компоненты, состояния).
- Явные экраны/роуты:
  - Queue,
  - Settings,
  - Updates,
  - About/Help.

### 4.2 Backend (Tauri/Rust)
- Tauri commands как boundary между UI и системным уровнем.
- Сервис-слой:
  - process manager для `aria2c`,
  - queue orchestration,
  - состояние задач,
  - логирование и события в UI.
- Event-driven поток:
  - backend публикует прогресс/статусы,
  - frontend подписывается и обновляет таблицу/индикаторы.

### 4.3 Хранение и безопасность
- Настройки: `tauri-plugin-store` (или эквивалентный storage).
- Секреты: системное защищенное хранилище (keyring/stronghold).
- По умолчанию секреты не сохраняются, только explicit opt-in.

### 4.4 Обновления
- Переход на штатный updater-механизм Tauri v2.
- Стандартизованный release pipeline под Tauri updater artifacts.
- Явный rollback/диагностика при неуспешном apply.

## 5. UI/UX целевого состояния

### 5.1 Принципы
- Современный визуальный язык (типографика, токены, доступность).
- Минимизация когнитивной нагрузки.
- Прозрачные статусы и диагностика без перегруза.

### 5.2 Предлагаемая структура
- Верхняя панель действий: добавление URL, быстрые действия очереди.
- Основная область:
  - таблица очереди (sticky header, статусы, массовые операции),
  - правая панель деталей выбранной задачи (опционально).
- Нижняя диагностическая панель:
  - прогресс сессии,
  - фильтруемые логи.
- Settings как отдельный экран/модалка с секциями:
  - Download,
  - Auth,
  - Network/Proxy,
  - Updates,
  - Language.
